# 输出范式全局标准

> 本仓库所有技能（SKILL.md）的输出格式遵循以下统一规范。
> 新技能或修改现有技能时，必须对齐本标准。

---

## 一、五大输出范式

| 范式 | 适用场景 | 格式要求 |
|:-----|:---------|:---------|
| **法条层级拆解** | 法律法规原文拆分 | 有序列表 + 层级缩进（篇/章/节/条/款/项/目） |
| **文书标准化** | 起诉状/答辩状/判决书/合同 | 标准公文段落 + 分区标题（当事人/诉求/事实/裁判） |
| **要点提炼** | 长文萃取/证据摘要/争议焦点 | 分项要点罗列 + 每条≤3 行 |
| **比对校验** | 新旧法条/合同版本/条款差异 | 双栏对照 `| 原条款 | 修订后 | 差异说明 |` |
| **结构化报告** | 法律意见书/评估报告/风险分析 | 固定章节模板（背景→分析→结论→建议） |

---

## 二、通用格式约束

### 2.1 标题层级
- `#` 仅用于文档标题（1 个）
- `##` 用于一级章节
- `###` 用于二级小节
- 禁止使用 `####` 及更深层级

### 2.2 法律引用格式
```
《中华人民共和国XX法》第X条第X款
```
- 法律名称使用书名号 `《》`
- 条/款/项/目使用中文数字（一、二、三）或阿拉伯数字（1、2、3），全文统一

### 2.3 表格规范
- 必须包含表头行 + 分隔行
- 对齐符：左对齐 `:-----`，居中 `:----:`
- 金额使用"元"为单位，精确到分



### 2.5 翻译腔禁令（句法 + 词汇双层）

> 翻译腔（Translationese）：语法无错但不符合中文母语者行文习惯的生硬表达。AI 逐 token 生成天然倾向词对词直译，必须显式约束。

#### 第一层：句法结构约束（强制）

**（一）禁用被动语态 — 用主动/无主语句替代**

| ❌ 翻译腔 | ✅ 地道中文 |
|:---------|:----------|
| 该合同被双方签署 | 双方已签署该合同 |
| 你的请求已经被批准 | 你的请求已获批 |
| 违约金被法院调整为 | 法院将违约金调整为 |
| 证据被提交至 | 证据已提交至 |

> 规则：全文"被"字使用次数 ≤2 次/千字。优先使用"由""经""已""将"替代。

**（二）拆分超长定语 — "的"字不过三**

| ❌ 翻译腔 | ✅ 地道中文 |
|:---------|:----------|
| 一个由我们的技术团队在过去三个月里为了解决用户隐私泄漏问题而开发出来的新系统 | 过去三个月，我们技术团队开发了这个新系统，专门解决用户隐私泄露问题 |

> 规则：单句"的"字使用 ≤3 个。超过 15 字的定语必须拆分为独立短句或状语前置。

**（三）省略冗余代词与量词 — 语境自明即省略**

| ❌ 翻译腔 | ✅ 地道中文 |
|:---------|:----------|
| 当他完成他的工作时，他露出了一个满意的笑容 | 工作完成后，他满意地笑了 |
| 原告提出了一个诉讼请求 | 原告提出诉讼请求 |
| 这是我们的一个核心观点 | 这是我方核心观点 |

> 规则：删去可省略的"一个""一项""他的""它的""他们的"。

**（四）隐化逻辑连词 — 意合优先于形合**

| ❌ 翻译腔 | ✅ 地道中文 |
|:---------|:----------|
| 当你到达法庭的时候，请保持安静 | 到达法庭请保持安静 |
| 因为合同已到期，所以被告应返还 | 合同已到期，被告应返还 |
| 如果发生违约，那么 | 若发生违约，则 |

> 规则：删除可省略的"当…的时候""因为…所以…""如果…那么…"（保留法律必需的条件句）。条件句用"若"替代"如果"。

#### 第一层补充：法律句法专有规则（学界共识）

**（五）名词化去臃肿 — 动词优先**

| ❌ 翻译腔（英文名词化直译） | ✅ 地道中文（动词化） |
|:---------|:----------|
| 合同的履行的完成 | 合同履行完毕 |
| 对合同条款的审查的进行 | 审查合同条款 |
| 违约责任的承担的实现 | 承担违约责任 |

> 规则：连续出现 3 个以上"的"连用时，检查是否为名词化直译，改为动词。

**（六）法律双联词/三联词 — 不逐词翻译**

英文法律惯用近义词并列以防遗漏（如"null and void""cease and desist""give, devise, and bequeath"），中文不需要逐词翻译。

| ❌ 翻译腔 | ✅ 地道中文 |
|:---------|:----------|
| 无效且不具效力（null and void） | 无效 |
| 停止并终止（cease and desist） | 停止 |
| 任何和所有权利（any and all） | 一切权利 |

**（七）"Shall"译为"应当"而非"将"**

| ❌ 翻译腔 | ✅ 地道中文 |
|:---------|:----------|
| 买方将支付价款 | 买方应当支付价款 |
| 双方将履行本合同 | 双方应当履行本合同 |

> 规则：义务性条款用"应当"或"须"，不得用"将"。

**（八）条件句前置 — "……的"字结构**

中文法律文书条件句必须前置，用"……的"字结构。

| ❌ 翻译腔 | ✅ 地道中文 |
|:---------|:----------|
| 甲方可解除合同，如果乙方逾期超过30日 | 乙方逾期超过30日的，甲方可解除合同 |
| 任何满足下列条件的人 | 符合下列条件者 |
| 合同自动终止，当许可期限届满时 | 许可期限届满的，合同自动终止 |

> 规则：条件在前、结果在后。"如果……那么……"→"……的，……"。

#### 第二层：法律专用词汇映射表

| ❌ 禁用（翻译腔/英美法） | ✅ 替换为（中国法言法语） |
|:---------|:------|
| Whereas / 鉴于（条款抬头） | 背景陈述 / 合同背景 |
| Hereby / 特此 | 据此 / 现（或直接省略） |
| Therein / 在其中 | 省略或"其中" |
| In Witness Whereof / 以资证明 | 删除，改为签章栏 |
| Subject to Section X | 根据第X条 / 除第X条另有规定外 |
| In the event of a breach | 若发生违约 / 发生违约情形时 |
| You are required to... | 你应当… / 请务必… |
| It is widely accepted that... | 普遍认为… / 业界公认… |
| Makes an attempt to... | 尝试… / 试图… |
| Indemnification / 赔偿保证 | 违约责任与损失赔偿 |
| Representation and Warranty / 陈述与保证 | 承诺与保证（中国法下不区分） |
| Best efforts / 最大努力 | 勤勉尽责 |
| Material adverse change / 重大不利变更 | 实质性不利变化 |
| Without prejudice to / 在不损害……的前提下 | 不影响……的情况下 |
| For the avoidance of doubt / 为免疑义 | 为明确起见 |
| Time is of the essence | 删除（中国法无此概念，直接约定逾期违约责任） |
| Governing law / 管辖法律 | 适用法律（中国境内合同不得选择外国法） |
| Counterparts / 签署份数 | 合同一式X份 |
| Severability / 可分割性 | 部分无效不影响其他条款效力（民法典第156条） |

> 全文不得出现左列词汇。违者视为翻译腔残留，需修正后重新输出。

#### 第三层：Prompt 工程约束（写技能时遵守）

编写或修改 SKILL.md 时，在系统指令中嵌入以下三句：

1. **断句**："将长句拆分为短句，避免使用超过 15 字的超长定语修饰名词。"
2. **语态**："多使用主动语态和无主语句，严格控制'被''所'等被动字眼的使用频率。"
3. **风格对齐**："输出前默念：'中国人不这么说话'——检查是否有词对词直译残留。"

## 三、各范式模板

### 3.1 法条层级拆解模板

```markdown
## 《法律名称》第X条

1. **第一款** — [原文]
   - （一）[项原文]
     - 1. [目原文]
2. **第二款** — [原文]
```

### 3.2 文书标准化模板

```markdown
## 当事人信息
- 原告：[名称/地址/法定代表人]
- 被告：[名称/地址/法定代表人]

## 诉讼请求
1. [请求一]
2. [请求二]

## 事实与理由
[段落叙述]

## 法律依据
- 《XX法》第X条：[适用理由]
```

### 3.3 要点提炼模板

```markdown
## 核心要点

1. **[关键词]** — [一句话概括，≤30 字]
2. **[关键词]** — [一句话概括，≤30 字]

## 详细分析

### [要点一]
[展开说明，≤3 行]
```

### 3.4 比对校验模板

```markdown
| 维度 | 原条款 | 修订后 | 差异说明 |
|:-----|:-------|:-------|:---------|
| 时效 | 3 年 | 5 年 | 延长 2 年 |
| 管辖 | 被告住所地 | 合同履行地 | 增加选择权 |
```

### 3.5 结构化报告模板

```markdown
## 一、案件背景
[简要事实]

## 二、法律分析
### 2.1 请求权基础
### 2.2 争议焦点

## 三、风险评估
| 风险项 | 等级 | 依据 |
|:-------|:----|:-----|

## 四、行动建议
1. [建议一]
2. [建议二]
```
