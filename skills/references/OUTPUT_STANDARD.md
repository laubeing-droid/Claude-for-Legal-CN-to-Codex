# 输出范式全局标准

> 本仓库所有技能（SKILL.md）的输出格式遵循以下统一规范。
> 新技能或修改现有技能时，必须对齐本标准。

---

## 一、五大输出范式

| 范式 | 适用场景 | 格式要求 |
|:-----|:---------|:---------|
| **法条层级拆解** | 法律法规原文拆分 | 有序列表 + 层级缩进（篇/章/节/条/款/项/目） |
| **文书标准化** | 起诉状/答辩状/判决书/合同 | 标准公文段落 + 分区标题（当事人/诉求/事实/裁判） |
| **要点提炼** | 长文萃取/证据摘要/争议焦点 | 分项要点罗列 + 每条≤3 行 |
| **比对校验** | 新旧法条/合同版本/条款差异 | 双栏对照 `| 原条款 | 修订后 | 差异说明 |` |
| **结构化报告** | 法律意见书/评估报告/风险分析 | 固定章节模板（背景→分析→结论→建议） |

---

## 二、通用格式约束

### 2.1 标题层级
- `#` 仅用于文档标题（1 个）
- `##` 用于一级章节
- `###` 用于二级小节
- 禁止使用 `####` 及更深层级

### 2.2 法律引用格式
```
《中华人民共和国XX法》第X条第X款
```
- 法律名称使用书名号 `《》`
- 条/款/项/目使用中文数字（一、二、三）或阿拉伯数字（1、2、3），全文统一

### 2.3 表格规范
- 必须包含表头行 + 分隔行
- 对齐符：左对齐 `:-----`，居中 `:----:`
- 金额使用"元"为单位，精确到分


### 2.5 翻译腔禁令词表

以下词汇为美国法/英美合同翻译腔，所有技能输出中**必须替换**为中国法对应表述：

| 禁止输出 | 替换为 |
|:---------|:------|
| Whereas / 鉴于（条款） | 背景陈述 / 合同背景 |
| Hereby / 特此 | 现（直接表述） |
| In Witness Whereof / 以资证明 | 删除，改为签章栏 |
| Indemnification / 赔偿保证 | 违约责任与损失赔偿 |
| Representation and Warranty / 陈述与保证 | 承诺与保证（中国法下不区分） |
| Best efforts / 最大努力 | 勤勉尽责 |
| Material adverse change / 重大不利变更 | 实质性不利变化 |
| Without prejudice to / 在不损害……的前提下 | 不影响……的情况下 |
| For the avoidance of doubt / 为免疑义 | 为明确起见 |
| Time is of the essence / 时间是合同的要素 | 删除（中国法无此概念，直接约定逾期违约责任） |
| Governing law / 管辖法律 | 适用法律（中国境内合同不得选择外国法） |
| Counterparts / 签署份数 | 合同一式X份 |
| Severability / 可分割性 | 部分无效不影响其他条款效力（民法典第156条） |

> ⛔ 生成合同时，全文不得出现上述左列词汇。违者视为翻译腔残留，需修正后重新输出。


## 三、各范式模板

### 3.1 法条层级拆解模板

```markdown
## 《法律名称》第X条

1. **第一款** — [原文]
   - （一）[项原文]
     - 1. [目原文]
2. **第二款** — [原文]
```

### 3.2 文书标准化模板

```markdown
## 当事人信息
- 原告：[名称/地址/法定代表人]
- 被告：[名称/地址/法定代表人]

## 诉讼请求
1. [请求一]
2. [请求二]

## 事实与理由
[段落叙述]

## 法律依据
- 《XX法》第X条：[适用理由]
```

### 3.3 要点提炼模板

```markdown
## 核心要点

1. **[关键词]** — [一句话概括，≤30 字]
2. **[关键词]** — [一句话概括，≤30 字]

## 详细分析

### [要点一]
[展开说明，≤3 行]
```

### 3.4 比对校验模板

```markdown
| 维度 | 原条款 | 修订后 | 差异说明 |
|:-----|:-------|:-------|:---------|
| 时效 | 3 年 | 5 年 | 延长 2 年 |
| 管辖 | 被告住所地 | 合同履行地 | 增加选择权 |
```

### 3.5 结构化报告模板

```markdown
## 一、案件背景
[简要事实]

## 二、法律分析
### 2.1 请求权基础
### 2.2 争议焦点

## 三、风险评估
| 风险项 | 等级 | 依据 |
|:-------|:----|:-----|

## 四、行动建议
1. [建议一]
2. [建议二]
```

---

## 四、合规检查清单

修改或新建技能时，逐项自检：

- [ ] 输出是否匹配五大范式之一？
- [ ] 法律引用格式是否统一（书名号 + 条款项）？
- [ ] 是否无英文/emoji/主观措辞？
- [ ] 表格是否有表头行 + 分隔行？
- [ ] 标题层级是否≤3 级？
